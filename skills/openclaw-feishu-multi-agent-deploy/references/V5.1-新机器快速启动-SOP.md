# V5.1 新机器快速启动 SOP

适用目标：

- 一台全新的 `Linux / macOS / Windows + WSL2` 电脑
- 需要把 `V5 Team Orchestrator / V5.1 Hardening` 从 0 拉起来
- 需要让客户第一次就按固定模板收集信息、生成配置、上线验证
- 目标效果：一个或多个飞书群，每个群固定 `1` 个 supervisor + `N` 个 worker，群与群隔离，worker 严格串行，主管统一收口

不适用目标：

- Windows 原生 service 路线
- 任意 mesh 式自由协作
- 同一个 bot 在同一个群里承担多个可见角色

---

## 0. 你先要搞清楚“哪台机器做什么”

推荐把机器分成两类：

1. 操作机（Operator Machine）
- 你本人使用 Codex 的电脑
- 用来安装这个 skill、准备输入文件、发 Codex 提示词、远程部署

2. 运行机（Gateway Machine）
- 真正运行 OpenClaw Gateway 的机器
- 真正连接飞书的机器
- 真正保存 `~/.openclaw/openclaw.json`、session、SQLite、manifest、watchdog 的机器

推荐生产实践：

- 你的笔记本 = 操作机
- 一台 Linux / WSL2 / macOS 机器 = 运行机

如果是小规模测试，也可以同一台机器同时承担两个角色。

本文默认：

- 你当前在操作机上
- 你要通过 SSH 部署到运行机

## 1. 部署前必须收集的真实信息

如果这些信息不齐，不要开始部署。

### 1.1 运行机信息

至少准备：

- 运行机地址或 SSH 别名
- SSH 用户名
- 是否是 `linux / macos / wsl2`
- 运行机上 `~/.openclaw` 是否允许写入
- 运行机是否已经安装过 OpenClaw

推荐先在操作机上配置一个 SSH 别名：

```sshconfig
Host openclaw-prod
  HostName 1.2.3.4
  User seaworld
  Port 22
```

这样后面所有命令都可以直接用：

```bash
ssh openclaw-prod
scp <本地文件> openclaw-prod:<远端路径>
```

### 1.2 每个飞书 bot 的应用信息

每个 bot 至少收集：

- `appId`
- `appSecret`
- `encryptKey`
- `verificationToken`
- 我们自己定义的 `accountId`

强约束：

- `accountId` 不是飞书自动给你的英文名
- `accountId` 是我们在 OpenClaw 配置里自己定义的稳定键名
- 推荐直接按角色命名：
  - `supervisor_bot`
  - `ops_bot`
  - `finance_bot`

### 1.3 每个目标群信息

每个群至少收集：

- 群名称
- 群真实 `peerId/chat_id`
- 是否要求必须 `@`
- 这个群启用哪些角色

如果你不知道群 `peerId`，先用下面的方法取：

1. 把 bot 拉进群
2. 群里 `@机器人` 发一条测试消息
3. 在运行机上看日志：

```bash
openclaw logs --follow
```

4. 从入站事件里找 `peer.id`
5. 群聊一般就是 `oc_xxx`

### 1.4 全局固定 bot-role 映射

先定死，不要边部署边改：

- 哪个 bot 永远是 `supervisor`
- 哪个 bot 永远是 `ops`
- 哪个 bot 永远是 `finance`

生产推荐：

- 同一个 bot 可以跨很多群复用
- 但它在所有群里都保持同一个角色

### 1.5 每个群的提示词和职责

每个群都要单独写：

- supervisor 的 `systemPrompt`
- 每个 worker 的 `systemPrompt`
- `description`
- `role`
- `responsibility`

不要偷懒把所有群写成同一段 prompt。可以复用骨架，但要保留群级差异。

## 2. 在运行机上安装 OpenClaw

以下命令基于 OpenClaw 官方文档当前写法整理，按 `2026-03-09` 验证。

### 2.1 基础要求

OpenClaw 官方当前要求：

- Node `22+`

先检查：

```bash
node --version
npm --version
```

如果 Node 不满足要求，先安装或升级 Node，再继续。

### 2.2 安装 OpenClaw

官方推荐快速安装：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

如果你习惯用 npm 全局安装，也可以：

```bash
npm install -g openclaw@latest
```

安装后先确认命令可用：

```bash
openclaw --version
openclaw doctor
```

如果提示 `openclaw: command not found`，优先修正 PATH：

```bash
export PATH="$(npm prefix -g)/bin:$PATH"
```

然后重新开一个 shell，再执行：

```bash
openclaw --version
```

### 2.3 安装 Gateway 服务

官方推荐：

```bash
openclaw onboard --install-daemon
```

这个命令会帮你完成：

- Gateway 服务安装
- 本地运行目录初始化
- 基础配置引导

安装完成后检查：

```bash
openclaw gateway status
openclaw status
openclaw dashboard
```

如果是 Linux / WSL2，也可以再确认 systemd user service：

```bash
systemctl --user status openclaw-gateway.service
```

如果是 macOS，确认 LaunchAgent 即可。

### 2.4 安装飞书插件

官方插件安装命令：

```bash
openclaw plugins install @openclaw/feishu
```

然后检查：

```bash
openclaw plugins list
openclaw plugins doctor
```

预期：

- 能看到 `@openclaw/feishu`
- 没有明显的 plugin load error

### 4.3 填写时最容易犯的错

部署前逐条确认：

- 不要把 `appId` 写到 `accountId`
- 不要让同一个 bot 在同一个群里承担两个可见角色
- 不要让 `workflow.stages` 漏掉 worker
- 不要让 `workflow.stages` 和 `workers[]` 不一致
- 不要把不同群写成同一个 `peerId`
- 不要把所有群的 `systemPrompt` 写成完全一样

## 8. V5.1 首次上线必须做的会话卫生

只要是下面这些场景之一，就先做 hygiene 再测：

- 第一次上线
- 大改 `systemPrompt`
- 大改 hidden main / callback 协议
- 从旧 V5 迁移到新 V5.1
- 之前群里已经出现过 `NO_REPLY / ACK_READY / COMPLETE_PACKET` 泄漏

内部群示例：

```bash
python3 ~/.openclaw/tools/v5/v51_team_orchestrator_hygiene.py \
  --home ~/.openclaw \
  --group-peer-id oc_f785e73d3c00954d4ccd5d49b63ef919 \
  --team-key internal_main \
  --include-workers \
  --delete-transcripts
```

外部群示例：

```bash
python3 ~/.openclaw/tools/v5/v51_team_orchestrator_hygiene.py \
  --home ~/.openclaw \
  --group-peer-id oc_7121d87961740dbd72bd8e50e48ba5e3 \
  --team-key external_main \
  --include-workers \
  --delete-transcripts
```

作用：

- 清理 supervisor `group/main`
- 清理 worker `group/main`
- 砍断旧协议 transcript 污染

## 9. 第一次真实验收怎么测

强规则：

- 一次只测一个群
- 先内部群，后外部群
- 只 `@supervisor`
- 一次发一整条任务，不要拆开发

### 9.1 内部群测试词

```text
@奥特曼 请为我们做一个“3天限时促销冲刺”执行方案。

目标：
1. 3天成交 100 单
2. 总预算不超过 10000 元
3. 综合毛利率不低于 35%
4. 明天上午 10 点前可以直接开跑

请按团队模式完成，不要只给概念建议，必须最终给出：
- 运营执行节奏
- 每天的动作安排
- 预算分配与 ROI / 毛利红线
- 风险预案
- 最终统一收口
- 明日三件事
```

### 9.2 外部群测试词

```text
@奥特曼 请为一个景区客户设计“节前 3 天拉新促销方案”。

约束：
1. 目标新增付费订单 120 单
2. 总预算不超过 15000 元
3. 综合 ROI 不低于 2.2
4. 综合毛利率不低于 30%
5. 要兼顾投放、私域、客服和履约风险

请按团队模式完成，最后必须给出：
- 运营推进方案
- 财务测算与红线
- 最终可直接执行的统一方案
- 明日三件事
- 风险预案
```

### 9.3 群内正确顺序

你应该看到：

1. `【主管已接单｜jobRef】`
2. `【运营进度｜jobRef】`
3. `【运营结论｜jobRef】`
4. `【财务进度｜jobRef】`
5. `【财务结论｜jobRef】`
6. `【主管最终统一收口｜jobRef】`

### 9.4 不应该出现的内容

- `ACK_READY`
- `REPLY_SKIP`
- `COMPLETE_PACKET`
- `WORKFLOW_INCOMPLETE`
- 长时间无回复
- 只跑一半不收口

如果群里已经出现 `jobRef`，运行机上立刻执行：

```bash

  --db ~/.openclaw/teams/internal_main/state/team_jobs.db \
  get-active \
  --group-peer-id oc_f785e73d3c00954d4ccd5d49b63ef919
```

或者外部群：

```bash
python3 ~/.openclaw/tools/v5/v51_team_orchestrator_runtime.py \
  --db ~/.openclaw/teams/external_main/state/team_jobs.db \
  get-active \
  --group-peer-id oc_7121d87961740dbd72bd8e50e48ba5e3
```

```bash
python3 ~/.openclaw/tools/v5/v51_team_orchestrator_runtime.py \
```

- `status=done`
- `active=null`
- `rollupVisibleSent=true`

## 11. 如果卡住了，优先怎么排查

### 11.1 先看运行健康

```bash
openclaw health
openclaw channels status --probe
```

### 11.2 再看 watchdog / reconcile

Linux / WSL2：

```bash
systemctl --user status v5-team-internal_main.service --no-pager -n 80
systemctl --user status v5-team-external_main.service --no-pager -n 80
systemctl --user list-timers --all 'v5-team-*' --no-pager
```

### 11.3 再手动补一次 resume-job

内部群：

```bash
python3 ~/.openclaw/tools/v5/v51_team_orchestrator_reconcile.py \
  --manifest ~/.openclaw/v5-runtime-manifest.json \
  --team-key internal_main \
  --openclaw-home ~/.openclaw \
  --openclaw-bin ~/.npm-global/bin/openclaw \
  resume-job \
  --stale-seconds 180
```

外部群：

```bash
python3 ~/.openclaw/tools/v5/v51_team_orchestrator_reconcile.py \
  --manifest ~/.openclaw/v5-runtime-manifest.json \
  --team-key external_main \
  --openclaw-home ~/.openclaw \
  --openclaw-bin ~/.npm-global/bin/openclaw \
  resume-job \
  --stale-seconds 180
```

### 11.4 这类现象说明什么

- supervisor 群第一条就裸 `NO_REPLY`
  - 先 hygiene，再 `resume-job`
- worker 裸 `NO_REPLY`
  - 看 `resume-job` 是否已经自动内联重派
- hidden main 里只有占位 `COMPLETE_PACKET`
  - 看新控制面是否已从 worker transcript 恢复真实 `messageId`
- DB `done` 但群里没最终收口
  - 看 `rollupVisibleSent` 是否为 `true`

## 12. 建议你保留的回滚命令

任何改动前先备份：

```bash
ts=$(date +%Y%m%d-%H%M%S)
mkdir -p ~/.openclaw/backups/$ts
cp ~/.openclaw/openclaw.json ~/.openclaw/backups/$ts/openclaw.json
cp ~/.openclaw/v5-runtime-manifest.json ~/.openclaw/backups/$ts/v5-runtime-manifest.json 2>/dev/null || true
```

如果这次上线有问题，最小回滚：

```bash
cp ~/.openclaw/backups/<时间戳>/openclaw.json ~/.openclaw/openclaw.json
cp ~/.openclaw/backups/<时间戳>/v5-runtime-manifest.json ~/.openclaw/v5-runtime-manifest.json 2>/dev/null || true
openclaw config validate
openclaw gateway restart
```

如果这次同时替换了 `~/.openclaw/tools/v5` 下的脚本，也要一并回滚。
